KE4H Project: Uzbek Food Knowledge Graph

This project explores the representation and enrichment of Uzbek food culture using Knowledge Graphs and Large Language Models.

Methodology

We started by identifying gaps in existing food-related knowledge graphs. We then used SPARQL to query Wikidata, and LLMs like ChatGPT to generate additional structured data. Finally, we transformed this data into RDF triples and visualized the results.

SPARQL Queries

# Example Query
SELECT DISTINCT ?foodLabel ?regionLabel WHERE {
  ?food wdt:P31 wd:Q2095 ;
        wdt:P495 wd:Q265 .
  OPTIONAL { ?food wdt:P495 ?region. }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en". }
}
LIMIT 10

LLM Prompts & RDF Triples

Prompt:

"Please describe the Uzbek dish plov in a structured way, listing ingredients, origin, cooking method, and any cultural notes."

LLM Output:

{
  "name": "Plov",
  "ingredients": ["rice", "carrot", "onion", "lamb", "vegetable oil", "cumin"],
  "region": "Samarkand",
  "cooking_method": "fried and steamed",
  "cultural_note": "Served at weddings and celebrations"
}

RDF Triples:

@prefix :  .

:plov a :UzbekDish ;
      :hasIngredient :rice, :carrot, :onion, :lamb, :vegetableOil, :cumin ;
      :originRegion :Samarkand ;
      :hasCookingMethod :friedAndSteamed ;
      :description "Served at weddings and celebrations" .

Discussion & Results

We observed that different LLMs provide varying degrees of structure and accuracy. RDF triples were manually verified, and some required adjustments to match ontological constraints. This method demonstrated the potential of LLMs in semantic enrichment.
