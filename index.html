KE4H Project: Uzbek Food Knowledge Graph

This project explores the representation and enrichment of Uzbek food culture using Knowledge Graphs and Large Language Models.

Methodology

Our project followed a structured pipeline combining Knowledge Graphs (KG), SPARQL, and Large Language Models (LLMs) to enrich and analyze data about traditional Uzbek foods.

Step 1: Topic Selection & KG Vocabulary

We selected ‚ÄúUzbek Foods‚Äù as our domain of interest. We identified relevant RDF classes and properties such as :UzbekDish, :hasIngredient, :originRegion, and :hasCookingMethod.

Step 2: SPARQL Queries

We designed multiple SPARQL queries to retrieve and explore existing data from Wikidata. These queries used key SPARQL clauses like: OPTIONAL, FILTER with REGEX, DISTINCT, LIMIT, UNION, and ORDER BY.

All SPARQL code was saved in the uzbek_food_queries.txt file.

Step 3: LLM-based Enrichment

We used multiple LLMs (ChatGPT and Gemini) to generate structured descriptions of Uzbek dishes like plov, somsa, and manti. The prompts asked the models to provide ingredients, origin, cooking methods, and cultural facts.

"Please describe the Uzbek dish plov in a structured way, listing ingredients, origin, cooking method, and any cultural notes."

The outputs were converted into RDF triples and saved in llm_outputs_and_triples.txt.

Step 4: RDF Triples

The triples were written in Turtle format (.ttl) and include custom vocabulary. We used prefixes like:

@prefix : <http://example.org/uzfood#> .

Each dish was modeled with relevant properties (e.g., :hasIngredient, :originRegion).

Step 5: Publishing the Project

We developed a GitHub Pages website to host our project. The site is organized into:

  • Project Overview
  • SPARQL Results
  • LLM Comparisons
  • RDF Enrichment
  • Final Reflections
